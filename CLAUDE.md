# MyTurnBaseGame — Claude 작업 지침 (레포 메모리 인덱스)

이 레포는 Unity(URP) **동시 계획 카드 턴제 1:1 택티컬 파이터**. 이 문서는 레포의 **기억 인덱스**다.
외부 작업 머신엔 별도 기획 메모가 없으므로, **Claude는 이 레포 + 작업 이슈만으로 맥락을 복원**한다.

## 🧭 먼저 읽을 것 (맥락 복원 순서)
1. `Docs/GDD.md` — 이 게임이 무엇인가(정체성).
2. 작업 중인 **GitHub 이슈**(자기완결) — 무엇을·왜·어떻게 끝내는지.
3. (있으면) `Docs/` 하위 **시스템 문서** — 전투/자원/연출 등 동작 스펙.

## 📁 Docs 구조
- `Docs/GDD.md` — 정체성(무엇인 게임).
- `Docs/combat-rules.md` — 전투 규칙(동작): 그리드·라운드/슬롯/비트·이동·공격·가드·승패·결정론 원칙. (E1)
- `Docs/combat-sim.md` — 결정론 전투 sim 아키텍처(`Assets/Game/Combat/Sim`): 상태 모델·RNG·이벤트 타임라인·resolver 계약. (E1 #11)
- 시스템 문서는 *각 작업 이슈가 필요한 만큼 추가/갱신*. 동작·수치는 GDD에서 분리.

## 🗂 작업 워크플로
- 작업 = **GitHub 이슈**(이 레포) + Projects 보드 추적. 이슈는 **자기완결**(이 레포만 보고 구현 가능)하게 작성.
- 코드/문서 변경 = **브랜치 + PR** (main 직접 커밋 금지). 브랜치명 예: `feature/<이슈번호>-<주제>`.
- 분류는 Projects 필드(영역/유형/Priority/Size/Estimate/Iteration). 큰 이슈는 **서브이슈 + Relationships** 매핑.
- **커밋 메시지**: 제목 `<유형>: <영역>-<주제> <#이슈번호 or 머지번호>` + **불렛 본문**(한국어).
  - `유형`: Feat / Fix / Docs / Refactor / Chore 등 (Projects '유형'과 정렬).
  - `영역`: E1~E8 등 에픽·영역 (Projects '영역'과 정렬).
  - 본문: 변경의 핵심(무엇을·왜)을 `- ` 불렛으로 나열. 관심사가 다르면 커밋을 분리.
  - 예:
    ```
    Feat: E7-임포트 파이프라인 셋업 #40

    - glTFast 패키지 추가 — glTF 임포트 경로 확립
    - ModelingTest 씬 신설 — 임포트 모델 툰 셰이더 검증용
    ```

## ✍️ 문서 규칙
- 레포 문서는 **자기완결**로 작성(외부 도구·외부 노트 경로 참조 금지).
- **GDD는 정체성만.** 동작·수치는 시스템 문서로 분리.
- 한국어. 모든 변경은 PR로.

## 🛠 기술 스택
