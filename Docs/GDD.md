# 🎮 GDD — MyTurnBaseGame (가제)

> **v0.1** · 이 문서는 **"이 게임이 무엇인가(정체성)"** 만 다룬다.
> 게임이 *어떻게 동작하는가*(메커니즘·수치)는 `Docs/` 하위 시스템 문서로 분리한다(예: `combat-rules.md`·`combat-sim.md`).

## 1. 한 줄 컨셉
상대의 다음 수를 읽어내는 **동시 계획 카드 턴제 1:1 택티컬 파이터**. 3D 서브컬쳐·툰 셰이딩의 스타일리시한 연출이 특징.

## 2. 장르 / 레퍼런스
- **장르**: 카드 기반 1:1 택티컬 파이터 (턴제).
- **영감(레퍼런스)**: 「이누야샤 데몬 토너먼트」류의 동시 계획 카드 전투. *(영감일 뿐 — 캐릭터·세계관은 오리지널로 제작.)*
- **무드 레퍼런스**: 페르소나 / 서브컬쳐(애니풍)의 감각적 스타일.

## 3. 핵심 재미 / 판타지
- **"상대의 3수를 읽어라."** 양측이 행동을 비공개로 정하고 동시에 부딪히는 **읽기 심리전**.
- 그 결과가 툰셰이딩 3D 액션으로 터지는 순간의 쾌감.

## 4. 톤 / 아트 방향
- **3D · 서브컬쳐(애니풍) · 툰 셰이더**.
- 스타일리시·팝 무드(페르소나풍): 감각적 카드 UI, 리듬감 있는 연출.
- 일부 캐릭터의 스킬 컷씬에서 **느와르(흑백)·팝아트** 스타일로 전환했다가 복귀하는 연출(이질감 없이).

## 5. 타겟 플랫폼 / 모드
- **플랫폼**: **PC 우선 → 이후 모바일 확장**.
- **모드**: v1.0.0은 **싱글 PvE(AI 상대)** 중심 → 이후 **PvP** 확장.

## 6. 범위 밖 (이 문서에서 다루지 않음)
- 전투 메커니즘·규칙 → `combat-rules.md`, 코드 구조·결정론 → `combat-sim.md`.
- 수치·밸런스(HP·데미지·아크량 등), 캐릭터별 데이터, 상세 UI/UX 사양.
