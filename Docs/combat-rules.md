# ⚔️ 전투 규칙 (E1 코어)

> 게임의 전투가 **어떻게 동작하는가**(메커니즘). 정체성은 `GDD.md`, 코드 구조·결정론은 `combat-sim.md`.
> 수치(HP·데미지·사거리·아크량 등)는 **밸런스 영역**이라 여기서 다루지 않는다.

## 그리드
- **3행 × 5열** 2D 그리드(좌표 기반, 칸 수는 추후 튜닝).
- 유닛 크기 = **1칸**. 단 칸은 **비배타** — 여러 유닛이 같은 칸에 겹칠 수 있다(스택 허용; 이동은 경계만 제약). **facing 미도입(MVP)** — 공격 패턴은 '상대(최근접 적) 방향' 자동 정렬(#14).

## 전투 규모
- **MVP = 1대1.** 단, 데이터·로직은 **n대n / 태깅(교대) 확장 가능**하게 설계 — 특정 유닛을 A/B로 하드코딩하지 않고 팀/유닛 리스트로 모델링.

## 라운드 / 슬롯 / 비트
- 한 라운드(턴) = **카드 3장(슬롯)**. 양측 **비공개 배치 → 동시 공개**.
- 해결: **슬롯 1 → 2 → 3 순차**, 각 슬롯은 **양측 동시**.
- 한 슬롯 내 처리 순서(비트): **이동 → 가드 → 공격 → 충전**.

## 이동
- 기본 **직교 1칸**(상하좌우). 대각·2칸·돌진은 고유 카드.

## 공격
- 카드별 **상대 패턴**(타격 셀, 유닛 기준 상대 좌표). 기준 방향 = **무조건 상대(최근접 적) 방향 자동 정렬**(facing 없음, #14) — 매 판정 시 현재 위치로 방향을 다시 잡아 유닛이 지나쳐도(p2-p1) 모호하지 않다.
- 명중 = 대상이 타격 셀에 존재. **스택이면 그 칸의 여러 대상 모두**(다중 명중), 여러 타격 셀 = 광역. 아군은 비명중.
- (구현 상세 — orientation 4방 회전·다중 명중은 `combat-sim.md`.)

## 동시 공격 우선도
- **Speed(유닛 스탯)** 높은 쪽 먼저.
- 임계치 내 동률 → **시드 RNG 랜덤**(결정론 유지).
- **선공 우선** — 빠른 쪽이 먼저 해결 → 대상이 HP≤0이면 그 유닛은 이후 공격 불가(반격 무효). Speed = 이니셔티브.

## 데미지 (구조, #20)
- **기본 → 가산 보정 → 곱연산 배율**: `최종 = (기본 + Σ가산) × Π곱연산`, 0 미만은 0.
- **가드는 뒷단**: 완전 가드 → 0, 기본 가드 → `max(0, 데미지 − 고정 N)`(고정 경감이라 약공에 강·강공에 약).
- 수치(기본·가산·배율·N)는 **밸런스(E3)** — 여기선 구조만. HP=float이라 데미지도 float.
- 구현 = `DamagePipeline`(효과 시스템 `DamageEffect`가 호출). 상세 `combat-sim.md`. HP 상한·회복은 E4/회복(이번 범위 밖).

## 자원 (요약)
- **아크(Arc)** = 정수 칸(유닛별 총량). 턴종료 +1, 기모으기 카드 +n, cap.
- **HP** = 실수(float)(유닛별). 데미지에 곱연산이 붙어 실수.
- **가드 2층**: 기본 가드(공용·고정 경감) / 완전 가드(고유·데미지 0).
- 상세 수치·공식은 자원/밸런스(E3) 영역.

## 승패
- **HP ≤ 0 → 패배.** 동시 0 → 무승부(엣지, 추후).

## 결정론 (원칙)
- 모든 무작위는 **시드 RNG**. 동일 입력 + 동일 시드 → 동일 결과(리플레이·PvP·RL 대비).
- sim 출력 = **결과 이벤트 타임라인**. 구현 상세는 `combat-sim.md`.
