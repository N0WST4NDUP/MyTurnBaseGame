# ⚔️ 전투 규칙 (E1 코어)

> 게임의 전투가 **어떻게 동작하는가**(메커니즘). 정체성은 `GDD.md`, 코드 구조·결정론은 `combat-sim.md`.
> 수치(HP·데미지·사거리·아크량 등)는 **밸런스 영역**이라 여기서 다루지 않는다.

## 그리드
- **3행 × 5열** 2D 그리드(좌표 기반, 칸 수는 추후 튜닝).
- 유닛은 1칸 점유. **facing 미도입(MVP)** — 공격 패턴은 '상대 방향' 기준.

## 전투 규모
- **MVP = 1대1.** 단, 데이터·로직은 **n대n / 태깅(교대) 확장 가능**하게 설계 — 특정 유닛을 A/B로 하드코딩하지 않고 팀/유닛 리스트로 모델링.

## 라운드 / 슬롯 / 비트
- 한 라운드(턴) = **카드 3장(슬롯)**. 양측 **비공개 배치 → 동시 공개**.
- 해결: **슬롯 1 → 2 → 3 순차**, 각 슬롯은 **양측 동시**.
- 한 슬롯 내 처리 순서(비트): **이동 → 가드 → 공격 → 충전**.

## 이동
- 기본 **직교 1칸**(상하좌우). 대각·2칸·돌진은 고유 카드.

## 공격
- 카드별 **상대 패턴**(타격 셀, 유닛 기준 상대 좌표). 패턴은 '상대 방향' 기준.
- 명중 = 대상이 타격 셀에 존재.

## 동시 공격 우선도
- **Speed(유닛 스탯)** 높은 쪽 먼저.
- 임계치 내 동률 → **시드 RNG 랜덤**(결정론 유지).

## 자원 (요약)
- **아크(Arc)** = 정수 칸(유닛별 총량). 턴종료 +1, 기모으기 카드 +n, cap.
- **HP** = 실수(float)(유닛별). 데미지에 곱연산이 붙어 실수.
- **가드 2층**: 기본 가드(공용·고정 경감) / 완전 가드(고유·데미지 0).
- 상세 수치·공식은 자원/밸런스(E3) 영역.

## 승패
- **HP ≤ 0 → 패배.** 동시 0 → 무승부(엣지, 추후).

## 결정론 (원칙)
- 모든 무작위는 **시드 RNG**. 동일 입력 + 동일 시드 → 동일 결과(리플레이·PvP·RL 대비).
- sim 출력 = **결과 이벤트 타임라인**. 구현 상세는 `combat-sim.md`.
